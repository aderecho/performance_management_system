# Skill: Planning

## Purpose
Create structured implementation plans before coding.

---

# Responsibilities

- Analyze requirements
- Break tasks into phases
- Identify risks and dependencies
- Define architecture direction
- Estimate implementation complexity
- Suggest scalable solutions

---

# Planning Workflow

## 1. Requirement Analysis

Always:
- understand the problem first
- identify business goals
- clarify constraints
- detect missing requirements

Output:
- concise requirement summary

---

## 2. Architecture Planning

Define:
- system structure
- module boundaries
- data flow
- API communication
- database relationships

Consider:
- scalability
- maintainability
- security
- performance

---

## 3. Task Breakdown

Break work into:
- small implementation steps
- independent modules
- testable milestones

Preferred format:

1. Setup
2. Core implementation
3. Validation
4. Testing
5. Optimization

---

## 4. Risk Assessment

Identify:
- technical risks
- scalability concerns
- integration complexity
- security issues

Provide:
- mitigation strategy
- fallback options

---

# Output Format

Use:
- checklists
- numbered phases
- concise explanations

Preferred:
- actionable tasks
- implementation-first planning

Avoid:
- vague suggestions
- overly abstract planning

---

# AI Agent Rules

Before planning:
- inspect existing architecture
- reuse established patterns
- avoid unnecessary complexity

Never:
- overengineer MVP features
- introduce unrelated technologies
- ignore current project standards
